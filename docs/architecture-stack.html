Agent-Gov-Sec — Layered Architecture

Two orthogonal, pluggable axes — framework (how an agent is orchestrated) and cloud (where identity / secrets / egress / tracing / audit resolve) — meeting at an agnostic core. Focus path highlighted: LangGraph + AWS.

How to read it: a request flows top→bottom. A payload persona is built by the selected framework adapter, which calls the shared GuardPipeline in the agnostic core at three hooks (before_model / after_model / before_tool). The core resolves identity, egress, tracing, and audit through the selected cloud adapter, which talks to the concrete cloud services. The two axes compose freely (e.g. --framework langgraph --aws); the governance is identical regardless of which framework or cloud is chosen. See architecture-framework-aws.md for the full write-up.
