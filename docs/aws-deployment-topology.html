How to read this. The top three layers (application, governance/orchestration, runtime) are largely cloud-agnostic — they live in core/, governance/, a2a/ and depend only on the interface seam. Only the bottom two layers (AWS platform services, observability) are provider-specific, supplied by cloud_adapters/aws/.

Status. This is the WS5 target topology. The AWS adapter is an interface-complete skeleton today (every accessor raises NotImplementedError); Azure is the one fully-implemented provider. The application layer reflects the current repo — a single Analyzer demonstration agent in payload_agents/; the full multi-agent migration product is archived.
